响应式交互

对操作的响应能够提高用户的使用信心并吸引他们。用户很期待在他们做出操作后，应用能够提供合理且优雅响应。这种响应须是有意义、可重复的，如清风拂面般舒适而不恼人。这种响应使得用户更加有兴趣深入探索应用：“如果触摸一下这里，会发生什么呢？”

用户输入 User input

在 material design 中，应用应当积极的响应用户输入：

  • 包括触摸、声音、鼠标和键盘等输入方式。
  • 虽然 UI 元素看起来是能够操作的，但是它们仍然处于一块屏幕中。因此需要通过视觉和触觉的反馈来消解这种隔阂感。

及时的响应能够提升应用的层次，使它不仅仅能够传递信息，更能够和用户进行优雅的交互。

表面的响应 Surface Reaction

当收到用户的输入，系统将立即做出视觉上的确认，提示用户已经做出了正确的操作，例如: 触摸了按钮、调整了音量或者按下了按键。这种设计就像墨水滴在纸面上的感觉，提供了持续的反馈。

触摸的涟漪 (Touch Ripple) 效果是这种视觉反馈的核心，它反映了触摸的位置、力度和相应的声音效果。


最佳实践

当一个输入事件发生在某个坐标时，应当从该坐标向外辐射出视觉效果。

Material 响应 Material Response

除了墨水似的表面响应，物体本身也应当能够响应操作。物体可以在触摸或点击时弹起，表明其激活状态。用户可以生成新的物体，或者调整物体，甚至拖动、翻转它们。

发生点

当新的物体作为响应出现在用户面前时，表面的变化应当从触摸点开始发生。

物体从触摸点逐渐出现，向用户反映其触摸操作的位置。

物体从屏幕中心出现，和用户的触摸操作似乎没有什么联系。


触摸时弹起

当卡片或者互相分割的元素被激活时，它应当弹起以表明其响应操作。

放射性的操作 Radial Action

所有的输入都有一个中心，也就是操作发生的坐标。为了清晰的反应这些操作，例如用户的触摸或者语音输入，屏幕上应当显示从中心发出的放射性效果，并且这种效果可以随着用户触摸的力度或者说话声音的变化而变化。

  • 触摸的中心是触摸发生的坐标，语音输入的中心是麦克风图标，键盘输入的中心是对应的按键。
  • 视觉反馈应当与用户的输入有联系，这些反馈均是从输入的中心发出，向外扩散，逐渐影响到周围的元素。
