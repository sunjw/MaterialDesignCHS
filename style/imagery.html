图像化

图像化的设计不仅仅是为了装饰，这种设计可以让您的应用与用户进行更好的交流。意图清晰的图像有助于引导用户的使用。

请参考以下的设计准则和最佳做法，使图像化的设计可以和您的品牌完美融合。

设计准则 Principles

当使用图像或照片来提升用户体验时，其仔细选择图片，体现与用户的关联、表达足够的信息并且保持美观。以下的设计准则可以帮助您设计产品。


与用户相关

与用户相关的图片有助于加强应用与用户的联系。

信息

通过图片，将特殊或者复杂的信息有效的表达给用户。

优雅

仔细斟酌的图片选择，让使用者感到您的应用是如此的优雅且贴近用户。


上下文相关

图像应当可以随着当前的逻辑而变化，智能的、上下文相关的图片可以进一步提高您应用的用户体验。


沉浸式的

在彩色的内容区域之下使用上下文相关的图片作为背景可以给用户沉浸于当前功能的感觉。


最佳做法 Best practices

合理使用多种媒体

抽象的图示和真实的照片都能用于产品之中。照片可以清晰的表达特殊的设置或者某个场景和故事。图示则可以表达某种概念或者对于一种场景的抽象。

对于指定的事务，可以选择照片来表达。

如果是某一类型的事务，可以选择使用图示，提高您应用在表示复杂内容时的效率。


不要使用素材图片

使用图像表达独特的效果和丰富的创造力。

对于特定的事务或者品牌内容，使用特定照片，对于某一类型的内容，使用抽象的图片。素材库的图片既不符合您的品牌也不能抽象的表达事务。

使用有故事的图片。

素材图片让人觉得死板。


 

特定的事务，使用特定的图片。

不要使用常见的素材。


吸引用户注意力

您选择的图像中应当有明确的重点，可以是某个事物，也可以是其它合理的东西，但请确保图片可以吸引用户注意力，且清晰的表达意图，使人过目不忘。

立即就能理解图像表达的意思。

不要让用户思索。


 

图像的意义通常仅需要几个聚焦的元素即可表达。

分散的焦点让用户南里抓住重点。


 

清晰到一看就能理解。

没有重点的图片没有意义。


说故事

让您的应用可以说出故事，仔细设计有意义的场景，并引导用户的感情。

有故事的图像能表达很多意义。

一般的图片不能很好的表达场景、功能和您的品牌。


 

合适的场景可以更有效的表达。

仅仅一个和上下文脱离的事务无法抓住用户注意力。


与 UI 集成 UI integration

合适的分辨率

仔细选择图片的大小，使其适合各种场景和平台。最好使用较大的图片，且不要让图片出现像素化的模糊。请在多种显示比例和不同的设备上进行测试。

合适的图片

被强行放大的图片


合适的比例

通过不同比例大小的图像配合，表达重点的内容。

在图册里，使用不同比例、大小的缩略图表达内容本身的区别。

在同一个系统中使用多种不同的框架。


保证文字的清晰

使用纹底保证文字的清晰可读。

深色的纹底根据背景的不同，透明度选择 20%-40%。浅色的纹底根据背景的不同，透明度选择 40%-60%。

纹底不要过度的虚化或覆盖背景图片。


 

对于大型的图片，使用专门的文字区域，而不要挡住图片。

过度虚化大型图片。


 

使用专门的色彩背景凸显文字，色彩背景的颜色根据内容进行选择，和图片并排显示，保证清晰的展现出文字内容。

您还可以使用品牌颜色来对图片进行处理。


头像和摘要

头像和摘要可以表示一条内容，它们可以是真实的照片或者抽象的图示。一般情况下点击头像或者摘要可以打开内容细节页面。

头像一般代表某个人，所以请提供个人化的选项，如果用户不选择进行设置，也请提供优雅的默认图片。有时也可以使用品牌 logo 作为头像。

摘要给用户提供主要内容的大概信息，您也可以在摘要中使用图片。

头像给用户个人化的感觉，还可以在很小的空间内表达足够多的信息。

摘要使用品牌作为头像，可以更好的表达内容大意，再加上图片，更加清晰。


Hero 图片

将 hero 图片放置在突出的位置，例如屏幕顶部的 banner。它们向用户提供当前内容的场景，或者用于突出品牌形象。

特色图像

特色图像突出当前内容，并放置在明显的位置。

Integrated hero image

嵌入的 hero 图像随着当前上下文的变化而变化，它们并不代表最重要的内容。


画廊

图像画廊通过整齐的图片表达一系列的内容。
